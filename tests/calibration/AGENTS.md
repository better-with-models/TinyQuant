# AGENTS.md — Guide for AI Agents Working in `tests/calibration`

This directory holds calibration tests that validate codec quality metrics against thresholds derived from real embedding data. Tests assert compression ratio, reconstruction fidelity (MSE / recall), determinism across runs, and alignment with the research baseline. The calibration fixture data is generated by `scripts/calibration/gen_openai_sample.py` (via `cargo xtask fixtures refresh-calibration`). CI runs these tests on a dedicated `rust-calibration` workflow. Changes here happen when codec quality thresholds are revised or new calibration scenarios are added.

## What this area contains

- primary responsibility: `test_compression_ratio.py`, `test_determinism.py`, `test_research_alignment.py`, `test_score_fidelity.py` — quality gate tests using pre-generated calibration fixtures; `conftest.py` for shared fixtures and fixture-path resolution
- main entrypoints: `conftest.py` (fixture loading), `test_score_fidelity.py` (recall and MSE thresholds)
- common changes: adjusting quality thresholds, adding new bit-width or dimension test cases, updating `conftest.py` when fixture paths change
