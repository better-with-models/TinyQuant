## Fixture strategy deviation from plan

The original plan assumed Python-generated binary fixtures for the codec pipeline.
This is **not possible** because Rust and Python use different RNG algorithms for the
rotation matrix:

- **Rust**: `ChaCha20Rng` + Box-Muller transform (established in Phase 13)
- **Python**: `numpy.random.default_rng` (PCG64) + `standard_normal`

Different RNGs produce different rotation matrices, which means different indices,
residuals, and decompressed values. The fixture approach was adapted:

- Binary fixtures (`*.bin`) are generated by the `dump_codec_fixture` Rust example
  binary (same pattern as Phase 13's `dump_rotation_fixture`).
- The `fidelity_manifest.json` is generated by Python тАФ it captures Python codec
  quality thresholds (MSE, Pearson ╧Б) for the fidelity gate.
- The "byte parity" test locks in Rust self-consistency (determinism) rather than
  cross-language parity.

**Lesson for future phases**: whenever a phase involves the full codec pipeline,
fixtures must come from Rust due to the rotation RNG divergence. Cross-language
byte parity is only achievable for components that do not touch the rotation matrix
(e.g., `compute_residual`, `Codebook::train`, `quantize_into`).

## Clippy constraints

The strict lint profile (`deny(indexing_slicing, cast_possible_truncation, unwrap_used,
expect_used)`) required narrow `#[allow(...)]` annotations in:

- `compressed_vector.rs`: `cast_possible_truncation` for `indices.len() as u32`
  (dimension bounded by u32 in practice)
- `residual.rs`: `indexing_slicing` for `chunks_exact(2)` chunk indexing (safe by
  `chunks_exact` contract)
- `service.rs`: `cast_possible_truncation` for `len() as u32`, `indexing_slicing` for
  the batch loop slice (bounds checked before the loop)
- `compressed_vector.rs`: `missing_const_for_fn` on `has_residual` тАФ `Option::is_some()`
  is not const-stable in MSRV 1.81 (stabilized in 1.87)

All `#[allow]` annotations are narrow (function- or block-level), never crate-wide.

## Observed fidelity numbers (Python codec, 1 000 vectors, bw in {2, 4, 8}, seed 42)

The Python codec with `residual_enabled=True` achieves near-perfect round-trip
(residual corrects quantization error in the rotated domain):

| bw | MSE max | Observed ╧Б |
|----|---------|------------|
| 2  | ~5e-8   | ~1.0       |
| 4  | ~9e-9   | ~1.0       |
| 8  | ~4e-9   | ~1.0       |

The fidelity gate in `fidelity_manifest.json` uses `rho_min = observed - 0.05 = 0.95`.
The Rust codec meets this gate because it applies the same residual correction math
in the rotated domain тАФ rotation only preconditions the quantization; the residual
fully compensates quantization error regardless of which orthogonal matrix is used.

## `service.rs` naming decision

The plan correctly identified the awkward `tinyquant_core::codec::codec::Codec` path
that would arise from naming the file `codec.rs`. Using `service.rs` keeps the internal
path `codec::service::Codec` while re-exporting as `tinyquant_core::codec::Codec`.
