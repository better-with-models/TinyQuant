# Behavior Specifications

> [!info] Purpose
> Use this folder for behavior-level requirements, acceptance criteria, and
> executable or near-executable specifications.

## Recommended contents

- Gherkin `.feature` files
- scenario catalogs
- testability notes
- links to related specs and design pages

## Conventions

- Keep behavior pages focused on observable outcomes
- Link supporting design decisions with wikilinks
- Update [[../index|the index]] when durable new pages are added
