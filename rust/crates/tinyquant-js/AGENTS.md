# AGENTS.md — Guide for AI Agents Working in `rust/crates/tinyquant-js`

This crate is the napi-rs binding layer that publishes TinyQuant as the
`@tinyquant/core` npm package. It wraps the `tinyquant-core` codec API and
exposes `CodecConfig`, `Codebook`, `Codec`, and `CompressedVector` to
JavaScript/TypeScript callers via N-API. Build output is a `.node` binary
consumed by the TypeScript fat-package (Phase 25).

## What this area contains

- primary responsibility: napi-rs N-API bindings for the `@tinyquant/core` npm package
- main entrypoints: `src/lib.rs` (napi module root), `build.rs` (napi-build script)
- common changes: adding new napi-exported types or methods when the core API evolves; bumping napi-rs or node-api-headers versions

## Common workflows

### Update existing behavior

1. Read `src/lib.rs` before adding or changing an exported type.
2. All exported structs must derive `#[napi]`; keep the JS surface minimal.
3. Run `cargo build -p tinyquant-js` and check the `.node` artifact compiles.
4. Update `package.json` version when the API changes.

### Add a new exported type

1. Add the struct or function to `src/lib.rs` with `#[napi]`.
2. Confirm the TypeScript declaration generated by napi-rs matches the intended signature.
3. Update the layout section if new source files are added.

## Invariants — Do Not Violate

- The `#![deny(missing_docs)]` lint is **not** enforced in this crate; keep public items documented with `///` comments regardless.
- Do not expose internal `tinyquant-core` types that are not part of the stable codec API.
- Semver: any removal or rename of an exported symbol is a breaking change to the npm package.
